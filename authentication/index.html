Authentication

How does authentication work with the BNF API?

BNF authentication is done through a HMAC signature which every request to BNF API needs to provide. Signing is done with a secret key that is obtained after creating a signalProvider in our system. There are two ways you are able to provide the signature, and it depends on the HTTP method used. For GET methods, signature is provided in the query string, as follows:

GET /balance?signature=SIGNATURE_HASH_HERE×tamp=TIMESTAMP_HERE&signalProvider=SIGNAL_PROVIDER_NAME_HERE for POST methods, the signature ends up in the payload of the request, as follows: POST /orders/create { ... signature: SIGNATURE_HASH_HERE, timestamp: TIMESTAMP_HERE, signalProvider: SIGNAL_PROVIDER_NAME_HERE }

Signature includes the entire payload (excluding itself), and is valid for 5 minutes. A detailed example on the workings of the authentication mechanism can be found in the BNF example implementation: Example implementation.
